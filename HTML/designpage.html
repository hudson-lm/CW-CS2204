Let and Const Basically let and var are really similar interchangeably. The reason I decided to use let over var, since let limits the variable declaration within its block. Using var can sometimes mistakenly declare two different variables with the same name, then accidentally replaced the old one. On the other hand, const is basically the keyword used to declare a variable that its value cannot be reassigned / changed to the new one after the first time declared. So, implementing const is also such a good practice to handle such values in a variable when we are sure we are not going to change it overtime and prevent accidentally changing it.

Next, I made the code for the text and video update(marked by comments), such as the index of each message and videos by accessing the element with id and querySelector (calling from the class_selector.js file). visit.js At first, I also covered all codes within the addEventListener() method. Next, I stated the onclick methods to both submit and reset button, then calling functions. In the submit button function (checkAvail(e)), I enter the parameter e(short for event), in order to declare e.preventDefault(). This is because once the button is clicked, the window won't get refreshed. In this function, I also called out the id and query selector, followed by setAttribute(), (from class_selector.js), in order to change the height of fieldset once the error message has been displayed. If there is no error, I am calling reserve function, provided default from CityU, to random the visitors available slot and return true or false.
For the reset onclick function, it just simply removes the innerHTML of error message and set back the height of fieldset to the beginning.
